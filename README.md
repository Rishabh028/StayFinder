## ✨ Key Features

StayFinder Pro is packed with features designed for users, property owners, and administrators.

-   🔐 **Secure User Authentication**: Robust sign-up and sign-in powered by Supabase.
-   🔍 **Dynamic Hotel Search**: A powerful and intuitive search interface to find the perfect stay.
-   🏨 **Detailed Hotel Pages**: Rich hotel profiles with photo galleries, amenities, and user reviews.
-   👤 **Personalized User Dashboard**: Track your booking history, nights stayed, loyalty points, and more.
-   📅 **Effortless Booking Flow**: A simple, multi-step process to book your room.
-   ❤️ **Wishlist Functionality**: Save your favorite hotels for later.
-   👑 **Admin Control Panel**: A comprehensive dashboard for admins to manage bookings, users, and site content.
-   🏠 **Owner Portal**: A dedicated space for hotel owners to manage their property listings and view analytics.
-   📱 **Fully Responsive Design**: A flawless experience on any device, from mobile phones to desktops.

## 🚀 Getting Started

Ready to run the project locally? Follow these simple steps.

### Prerequisites

-   Node.js (v18.x or later)
-   npm (or your favorite package manager)
-   A [Supabase](https://supabase.com/) account for API keys.

### Local Installation

1.  **Clone the Repository**
    ```sh
    git clone https://github.com/Rishabh028/StayFinder.git
    cd StayFinder
    ```

2.  **Install Dependencies**
    ```sh
    npm install
    ```

3.  **Set Up Environment Variables**
    Create a `.env` file in the project root. Copy the contents of `.env.example` (if available) or use the structure below. Get your keys from your Supabase project dashboard (`Settings > API`).
    ```env
    # .env
    VITE_SUPABASE_URL="YOUR_SUPABASE_URL"
    VITE_SUPABASE_ANON_KEY="YOUR_SUPABASE_ANON_KEY"
    ```

4.  **Launch the Development Server**
    ```sh
    npm run dev
    ```
    Open your browser and navigate to `http://localhost:5173`. You're all set!

## 📂 Project Structure

The project uses a feature-based structure to keep the code organized, scalable, and easy to maintain.

```
/src
├── components/     # 🌍 Global, reusable components (Button, Header, etc.)
├── context/        # 🔄 React Context for global state (e.g., AuthContext)
├── lib/            # 📚 External library configurations (supabaseClient.js)
├── pages/          # 📄 Top-level page components & their specific sub-components
│   ├── sign-in/
│   ├── user-dashboard/
│   └── ...
├── styles/         # 🎨 Global styles and Tailwind configuration
├── utils/          # 🛠️ Utility functions and helpers
├── App.jsx         # 🏁 Main application component
├── index.jsx       # 🎬 Application entry point
└── Routes.jsx      # 🗺️ All application routes
```

## ☁️ Deployment

This application is proudly deployed on **Vercel**, taking advantage of its seamless Git integration for continuous deployment.

-   **Trigger**: A new deployment is automatically initiated on every `git push` to the `main` branch.
-   **Build Command**: `npm run build`
-   **Output Directory**: `build/`
-   **Environment Variables**: Supabase keys are securely stored in the Vercel project settings.
